Built with <details name="faq">, ::details-content, and interpolate-size: allow-keywords — zero JavaScript, smooth height transitions.

🎯 What makes this accordion native?

The name attribute on <details> creates a mutual-exclusion group: when one opens, all others with the same name close automatically — no JavaScript.

The ::details-content pseudo-element exposes the hidden content block so CSS can target and animate it. Pair with interpolate-size: allow-keywords to finally animate height: 0 → auto.

details[name] ::details-content No JS
⚡ How does height: auto animation work?

Before (hacky)

max-height: 9999px trick — ugly easing, animates even when content is short.

Now (native)

interpolate-size: allow-keywords → browser measures and interpolates to the real height.

::details-content {
  height: 0;
  overflow: hidden;
  transition: height .45s ease;
}
details[open]::details-content {
  height: auto; /* 🎉 */
}

♿ Is it accessible by default?

Yes — completely. <details>/<summary> carry implicit ARIA roles. Screen readers announce open/closed state automatically. Keyboard: Enter or Space toggles.

No need for aria-expanded, aria-controls, or role="region" — the browser semantics handle it natively. Add @media (prefers-reduced-motion) to remove the transition for sensitive users.

🌐 Browser support today?

details[name] and ::details-content are Baseline Newly Available (Sept 2025) — Chrome, Firefox, and Safari all ship them. The smooth height animation (interpolate-size: allow-keywords) is Chromium-first.
