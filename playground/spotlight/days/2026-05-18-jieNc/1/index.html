animation-timeline: view()

Each card triggers its own animation when it enters the viewport.

🎞

view() timeline

Links an element's animation to its own scroll-into-view progress. No intersection observer needed.

📏

animation-range

Control exactly when the animation starts and ends within the entry / exit range of the viewport.

⚡

Zero JS

These keyframes are pure CSS. The browser handles all timing on the compositor thread — buttery smooth.

🎭

animation-fill-mode

Set animation-fill-mode: both to hold the final state after the animation completes.

🔗

Chaining

Multiple animations can share the same timeline, each with different ranges and delays.

📱

Respects prefers-reduced-motion

Wrap your scroll animations in @media (prefers-reduced-motion: no-preference) for accessibility.
