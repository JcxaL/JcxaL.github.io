Native <dialog> Animations

Three dialog variants — modal, bottom sheet, side drawer — all animated with @starting-style and transition-behavior: allow-discrete. Zero JS animation libraries.

Side Drawer

How it works: The <dialog> element transitions between display: none and display: block. CSS can't normally transition display, but transition-behavior: allow-discrete opts in. @starting-style defines the first-frame state so the enter animation plays. The exit animation plays during the closing phase before display: none takes effect.

The key CSS pattern

dialog {
  opacity: 0;
  transform: scale(.93) translateY(16px);
  transition:
    opacity .3s ease,
    transform .3s cubic-bezier(.34, 1.56, .64, 1),
    overlay .3s allow-discrete,
    display .3s allow-discrete;    /* allow-discrete lets display participate in transitions */
}

dialog[open] {
  opacity: 1;
  transform: none;
}

@starting-style {
  dialog[open] {    /* first-frame enter state */
    opacity: 0;
    transform: scale(.93) translateY(16px);
  }
}
Centered Modal

This dialog uses scale + translateY for a natural pop-in feel. The spring-like cubic-bezier (.34, 1.56, .64, 1) gives it a small overshoot.

Click the backdrop or ✕ to close. Notice the exit animation — it runs because transition-behavior: allow-discrete delays the display: none until the transition completes.
Bottom Sheet

Slides up from the bottom — a common mobile pattern now achievable with pure HTML + CSS.

The sheet fills full width thanks to margin: auto 0 0 combined with the dialog's default centering being overridden. border-radius: 20px 20px 0 0 gives it the native bottom-sheet feel.
Navigation

A side drawer using margin: 0 0 0 auto to push the dialog to the right edge.
