interpolate-size: allow-keywords — height:auto transitions

⚠️ interpolate-size may not be supported — Chrome 129+ or Firefox 130+ required. Transitions may be instant.

1 · Accordion (pure CSS once interpolate-size is set)

interpolate-size: allow-keywords is a new CSS property (2025) that lets you animate between a length value (0) and an intrinsic keyword like auto, min-content, or max-content. Previously this required JavaScript to measure the element's scrollHeight and set it explicitly.
The browser now knows how to interpolate between a concrete pixel value and a keyword. When transitioning from height: 0 to height: auto, it calculates the resolved auto value as the target and animates the pixel value smoothly. Setting it on the :root cascades everywhere on the page.
Chrome 129+ (September 2024), Firefox 130+, Safari 18.2+. As of mid-2026, global support is around 92%. The fallback (no transition, instant show/hide) is perfectly acceptable — use @supports (interpolate-size: allow-keywords) for a progressive enhancement approach.
Any size property! width: auto, height: min-content, block-size: max-content. It also works with calc() expressions that mix lengths and keywords. Combined with transition-behavior: allow-discrete you can now animate display: none ↔ block in the same rule — making pure-CSS reveal patterns finally practical.

3 · Before vs After

❌ Old way (JS required)

// Measure then set:
el.style.height = '0';
el.getBoundingClientRect();
el.style.height =
  el.scrollHeight + 'px';

// After transition:
el.style.height = '';

✅ New way (CSS only)

/* One rule on :root */
:root {
  interpolate-size:
    allow-keywords;
}
.panel {
  height: 0;
  transition: height .35s;
}
.open .panel {
  height: auto; /* 🎉 */
}
Transition speed 380ms
