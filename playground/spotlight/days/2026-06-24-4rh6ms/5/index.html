How CSS Scroll-Driven Animations Replace 100 Lines of JavaScript

Jun 24, 2026
8 min read
CSS, Animation, Performance

For years, progress indicators required JavaScript: attach a scroll listener, compute a ratio, set a style property. Now with CSS animation-timeline: scroll(), you can eliminate that entire pattern and hand the work to the browser compositor thread.

"The browser already knows the scroll position. You were just asking JavaScript to relay a message it was never meant to carry."

The Old Pattern

Dozens of Medium articles taught us to do this: window.addEventListener('scroll', () => { const pct = scrollY / (docH - winH); bar.style.transform = \`scaleX(\${pct})\`; }). This fires on the main thread, blocks rendering, and forces style recalculation every frame. On low-end devices it visibly stutters.

Developers eventually learned to requestAnimationFrame it, then to throttle with IntersectionObserver, then to debounce — layers of complexity to compensate for using the wrong tool.

The New Pattern

CSS animation-timeline: scroll() ties an @keyframes animation's progress directly to the document's scroll position — on the compositor thread, no JS required. The ring on the bottom-right and the bar at the top of this page are implemented in under 15 lines of CSS each.

For the linear bar: transform: scaleX(0) at 0% scroll, transform: scaleX(1) at 100%. For the SVG ring: stroke-dashoffset animates from 163.4 (full circle circumference, 2π×r) to 0.

Why It Matters

Performance is only part of the story. The real win is declarative intent: the CSS says "this property tracks scroll" and the browser handles when, where, and how. No event listeners to clean up, no race conditions, no scroll-jank from main-thread contention.

Combined with @scroll-timeline named timelines and the view() function for element-level triggers, scroll-driven animations unlock a new tier of scroll choreography that was previously only achievable with heavyweight libraries like GSAP ScrollTrigger.

Browser support in 2026: Chrome 115+, Firefox 110+, Safari 18+. Progressive enhancement friendly — the page works without animation on older browsers.

The Scroll Progress Ring

The circular ring uses an SVG <circle> with stroke-dasharray set to the circumference (2π × radius = 163.4 for r=26). The stroke-dashoffset starts at 163.4 (fully hidden) and animates to 0 (fully visible). The animation's progress is linked to scroll(), so the ring fills exactly as fast as you scroll.

The gradient stroke is achieved with a <linearGradient> referenced in the stroke attribute — purple → cyan → pink — giving the ring its distinct multi-color look even though it's a single SVG element.

What's Next

CSS scroll-state() — landing in browsers mid-2026 — extends this further. You'll be able to apply styles conditionally based on whether an element is currently stuck, overflowing, or snapped. The long-standing JavaScript scroll listener is rapidly becoming a legacy pattern.
