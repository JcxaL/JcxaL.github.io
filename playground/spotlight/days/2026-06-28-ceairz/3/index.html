Font Weight
on Scroll

CSS animation-timeline: view() drives font-variation-settings — text grows from hairline to heavy as it enters the viewport. Pure CSS, zero JS.

Variable fonts + scroll-driven Momentum

Each heading starts at weight 100 (hairline) and thickens to 900 (ultra-black) as the element scrolls into view — driven entirely by CSS animation-timeline: view() and @keyframes on font-variation-settings.

animation-range: entry Gravity

The animation-range: entry 20% entry 70% defines exactly when the animation plays within the view timeline — from when 20% of the element is visible to 70%.

Scroll as a timeline axis Resonance

Unlike scroll-triggered (on/off), scroll-driven is continuous — the font weight tracks your exact scroll position, pausing when you stop, reversing when you scroll back.

No JavaScript required Velocity

The entire effect is self-contained CSS. No IntersectionObserver, no GSAP, no event listeners. The browser's compositor handles it directly off the main thread.

Combine with other axes Amplitude

Variable fonts can animate wdth (width), opsz (optical size), slnt (slant), and custom axes — each independently drivable by scroll.

.heading {
  animation: thicken linear both;
  animation-timeline: view();
  animation-range: entry 20% entry 70%;
}

@keyframes thicken {
  from { font-variation-settings: 'wght' 100 }
  to   { font-variation-settings: 'wght' 900 }
}
