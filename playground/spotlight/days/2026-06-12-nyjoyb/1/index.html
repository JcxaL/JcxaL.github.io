Scroll-Driven Animations

Using animation-timeline: view() and animation-range: entry, each card triggers its own entrance the moment it enters the viewport — no JavaScript required.

Performance · Zero JS

Pure CSS, No Observers

Traditional scroll animations require IntersectionObserver + classList manipulation. The new View Timeline API offloads all of this to the compositor thread — silky smooth at 60fps with zero main-thread cost.

Technique · 3D Perspective

Rotated Entrance with Blur

The keyframe starts at rotateX(30deg) with a subtle blur(4px) and scale(0.94), then resolves to identity. This triple-axis entrance reads as both "arriving from depth" and "coming into focus".

Customization · Ranges

Fine-Tuned Entry Range

animation-range: entry 0% entry 40% means the animation completes in the first 40% of the element's journey through the viewport — the rest of the scroll feels instant and settled.

Variant · Exit Animation

Extend to Exit

Add a second keyframe block using animation-range: exit 60% exit 100% to fade cards out as they leave the top of the screen — creating a full "enter → linger → exit" choreography from pure CSS.
