Drop 1 / Technique

animation-timeline: view()

Links an animation's progress to an element's position in the scroll viewport. When the element enters view, the animation plays — no IntersectionObserver, no requestAnimationFrame, no JS at all.

Pure CSS
Drop 1 / Browser support

Chrome 115 · Edge 115 · Safari 18

As of 2026, scroll-driven animations are universally supported. Chrome 145 added scroll-triggered animations — discrete transitions that fire once when crossing a scroll threshold.

85%+ global
Drop 1 / animation-range

Precise Entry Windows

Use animation-range: entry 0% entry 35% to control exactly when during the element's entry the animation should play. Odd/even children get slightly offset ranges for a natural stagger.

Stagger
Drop 1 / Performance

Compositor-Thread Only

Scroll-driven animations that only animate opacity, transform, and filter run entirely off the main thread. The browser compositor handles them — no layout, no paint, no jank.

60fps guaranteed
