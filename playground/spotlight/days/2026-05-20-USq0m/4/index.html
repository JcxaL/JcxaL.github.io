The mechanism: While dragging, the card's translate + rotateX/Y is set directly via JS for immediate feedback. The rubber-band effect is simulated by dividing displacement by a resistance factor, so the card "fights back" as it moves further from its resting position. On release, the snapping class applies a CSS transition using a hand-crafted linear() timing function that encodes overshoot and settle — the browser GPU does the rest.
