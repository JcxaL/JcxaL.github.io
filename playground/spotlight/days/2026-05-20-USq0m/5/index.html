CSS Anchor Positioning: Each option gets an anchor-name. The pill uses left: anchor(--opt-N left) and width: calc(anchor(--opt-N right) - anchor(--opt-N left)) to perfectly match the active item's bounds — no JavaScript measurement needed in supporting browsers. A linear() spring timing function on left + width produces the characteristic iOS overshoot without any JS animation loop. The JS fallback uses getBoundingClientRect() for the same effect everywhere.
