scroll-state(stuck)

A new CSS container query that detects when a sticky element has become "stuck" — no JavaScript, no IntersectionObserver hacks.

↓

The Problem, Solved

For years, making a sticky navbar shrink when you scroll required sentinel elements, IntersectionObserver, or scroll event listeners. The new scroll-state() CSS container query changes that.

Wrap your sticky element in a scroll-state container, then write @container scroll-state(stuck: top) rules to style it when it's pinned to the viewport.

No JS required

Pure CSS detects the stuck state — zero JavaScript needed.

Smooth transitions

CSS transitions animate between normal and stuck states seamlessly.

Chrome 137+

Latest Chromium builds support scroll-state container queries natively.

How It Works

Scroll up to watch the nav compact — size, padding, blur and border all transition via CSS alone.

The key CSS: set container-type: scroll-state on the outer wrapper, then target it with @container scroll-state(stuck: top).

Beyond Navigation

scroll-state() works for table headers, sidebar sections, floating action buttons — any sticky element that needs to visually respond to its pinned state.

Table headers

Add background/shadow when header row sticks to keep it readable.

Sidebar sections

Highlight active section in a sticky sidebar as you scroll past it.

Snapped items

scroll-state(snapped: x) styles scroll-snap targets when centered in view.
