UI Spotlight · Day 22

Fluid Wave, :has() Grid,
Neon Flicker, Rubber Band, Expand Cards

Five fresh UI patterns: SVG fluid wave morph, CSS :has() sibling reactor, neon sign flicker, rubber band elastic drag, and animated expandable cards with interpolate-size.

Drop 1 ✨ Animation
SVG Fluid Wave Morph
Three layered SVG wave paths animated by morphing 80 bezier sample points each frame, with four presets and live sliders.
buildWavePath() computes point positions using summed sines at two different frequencies, generating a new SVG d string every rAF. Three waves with offset phases and damped amplitudes layer to create fluid depth — no SVG SMIL required.

Drop 2 ⬛ Layout
CSS :has() Sibling Reactor
An 18-cell emoji grid reacts to hover with zero JavaScript — hovered cells enlarge and glow while siblings dim and shrink. A second demo shows :has()-driven form validation labels.
.reactor-grid:has(.r-cell:hover) .r-cell:not(:hover) selects all non-hovered siblings whenever any cell is hovered — true CSS parent awareness. The form demo uses :has(input:valid:not(:placeholder-shown)) to turn labels green or red with no JS.

Drop 3 🔤 Typography
Neon Sign Flicker
Five neon signs with per-letter staggered flicker animations. Click to kill individual tubes, hover to buzz, trigger random faults, or hit Full Blackout.
Each letter is wrapped in a <span> with randomised animation-duration and animation-delay so timing is naturally unsynchronised. Four layered text-shadow values at increasing radii simulate neon tube glow. Dead letters get color:rgba(255,255,255,.08) and animation:none.

Drop 4 🧲 Micro-interaction
Rubber Band Elastic Drag
Six draggable cards stretch up to a clamped max distance from their resting position, then spring back with configurable stiffness, damping, and overshoot.
getBoundingClientRect() captures each item's resting centre. Displacement is clamped to maxStr during drag. On release, the spring loop applies force = -stiffness × x each frame, scaled by damping, until position and velocity fall below 0.1 px. A dashed SVG line shows the tether during drag.

Drop 5 🧩 Component
Expandable Cards — interpolate-size
An accordion of four cards that expand and collapse with a smooth height: 0 → height: auto CSS transition — no JavaScript height measurement needed.
:root { interpolate-size: allow-keywords } tells the browser to interpolate between a fixed length and an intrinsic keyword like auto. JS only toggles an .open class. A max-height fallback is applied for non-supporting browsers, detected with CSS.supports('interpolate-size', 'allow-keywords').
