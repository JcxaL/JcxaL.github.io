Scroll-Driven 3D Entrance

Pure CSS — no JavaScript. Each card enters from a unique 3D angle as it scrolls into view.

⚠ Your browser doesn't support animation-timeline: view() yet. Try Chrome 115+ or Firefox 110+ with the flag enabled.
🌌
Animation

Scroll-Driven Animations

CSS animation-timeline: view() links an element's animation progress directly to its position in the scrollport — no JS needed.

🎭
CSS Spec

Multiple Entry Directions

Each card in this demo has a different @keyframes rule — rotating in from left, right, above, or below — assigned via :nth-child.

🌿
Range Control

animation-range

The entry 5% entry 65% range means: start animating when 5% of the card enters the viewport, finish when 65% has entered.

🔥
Performance

Zero JavaScript

Because it's driven by the browser's scroll compositor, this technique runs at 60 fps even on slow devices — no scroll event listener needed.

💜
Easing

cubic-bezier Spring Feel

The cubic-bezier(.16,1,.3,1) easing — borrowed from the Radix spring curve — overshoots slightly to give a bouncy, physical feel.

🔷
3D Depth

Perspective + rotateY

Combining perspective(900px), rotateY(±35deg), and scale(.82) creates a convincing sense that cards are tumbling in from behind the screen.

✨
Compatibility

Progressive Enhancement

A @supports not (animation-timeline: view()) block shows a warning banner on unsupported browsers while keeping cards fully visible.

💖
Mobile

Touch-Scroll Works Too

The timeline responds to any scrolling mechanism — wheel, touch, keyboard — because it hooks into the browser's native scroll engine.

🌊
Remix

Mix & Match Directions

Try assigning enterU / enterD (rotating on the X axis) to alternate cards for a staircase-like reveal choreography.
