Zero-JS Animated Accordion

Plain native <details>/<summary> — no JS at all. The expandable region is styled directly with the ::details-content pseudo-element and animated to its true intrinsic height via height: calc-size(auto, size), made possible by the new interpolate-size property. No wrapper <div>, no max-height guessing games.

What makes this different from grid-template-rows tricks?
Older height-auto workarounds animate grid-template-rows: 0fr → 1fr on a wrapper, which works but requires an extra element and only approximates the real height. calc-size(auto, size) asks the browser to compute the box's actual intrinsic size and transition to that number — the real thing, not an approximation.
Does this need a JavaScript click handler?
No. <details> has built-in open/close behavior and keyboard support (Enter/Space on a focused summary) for free. The only CSS-only catch is that ::details-content and calc-size() are progressive enhancements — unsupported browsers just snap open/closed instantly, which is still fully usable.
What draws the rotating chevron?
A ::after pseudo-element on summary — a small rotated border-corner — instead of the default ::marker triangle, so it can be independently transitioned with transform: rotate() when the [open] attribute toggles.
Can I nest more content inside?
Yes — anything goes inside the content div: paragraphs, lists, images, even another accordion. Because the height is computed from real layout, there's no manual re-measuring needed if the content changes size later.
