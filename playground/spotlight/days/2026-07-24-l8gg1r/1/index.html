Trigger

Scroll Trigger & Reverse

Each card below plays a keyframe-style transition once when it crosses a trigger line — then reverses if you scroll back past it. Not continuous scroll-scrubbing; a discrete play/reverse, like a light switch tied to position.

↓ Scroll slowly to see each card trigger

Slide + fade

translateY + opacity

The simplest recipe: rises 46px while fading in. Scroll back up past the trigger line and it retreats the same way it arrived.

Clip-path wipe

inset() reveal

A clip-path: inset() mask wipes open left-to-right. No opacity change at all — the content is simply unmasked.

Scale pop

Springy scale-in

Scales up from 0.82 with an overshoot easing curve for a springy pop, rather than a linear ease.

Blur focus

Depth-of-field rack focus

Starts blurred and slightly offset, then racks into sharp focus — like a camera pulling focus onto the card.

Inspired by Chrome for Developers — "CSS Scroll-Triggered Animations" (animation-trigger / timeline-trigger) · UI Spotlight Day 68 · Drop #1
Reimplemented with IntersectionObserver + CSS transitions for broad compatibility today; the native CSS trigger property is the same discrete play/reverse idea, declared without JS.
