interpolate-size: allow-keywords

Smooth height:auto transitions — no JavaScript height tricks, no fixed pixel values.

Expanding FAQ

What is interpolate-size? ▼
A CSS property that tells the browser to interpolate between a fixed length and an intrinsic keyword like auto, min-content, or max-content — enabling smooth transitions to natural content sizes.
Why couldn't we animate to height:auto before? ▼
CSS transitions interpolate between two numeric values. auto is a keyword, not a number, so the browser couldn't compute an intermediate value. The old workarounds were JS-computed heights or the grid-template-rows: 0fr/1fr trick.
What browsers support it? ▼
Chrome 129+, Edge 129+, Safari 18+, and Firefox 131+ all support interpolate-size: allow-keywords. Set it on :root to enable it globally for the whole page. The fallback is an instant (no-animation) jump — still functional.
Does it work on width too? ▼
Yes! It works on width, height, block-size, inline-size, and shorthand properties. Animating to min-content or fit-content is now possible too.

Expandable Code Preview

styles.css click to expand ▼ 
/* The magic one-liner */
:root {
  interpolate-size: allow-keywords;
}

/* Now height:auto transitions work! */
.panel {
  height: 0;
  overflow: hidden;
  transition: height 0.4s ease;
}

.panel.open {
  height: auto; /* Smooth! */
}

/* Also works with min-content, max-content */
.chip {
  width: 40px;
  transition: width 0.3s;
}
.chip:hover {
  width: max-content;
}
