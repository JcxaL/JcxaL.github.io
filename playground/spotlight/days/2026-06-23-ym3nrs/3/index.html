CSS Custom Highlights API

The CSS Custom Highlights API, available since Chrome 105 and Firefox 119, lets developers highlight arbitrary text ranges without modifying the DOM. Unlike mark elements or wrapper spans, highlights are purely presentational — they live in a separate layer and don't affect layout or accessibility trees.

Using CSS.highlights.set(), you register named highlight objects against AbstractRange instances. The stylesheet then targets them via the ::highlight(name) pseudo-element, supporting color, background-color, text-decoration, and a handful of other inherited properties.

This technique shines in editors and readers where "find in page", grammar checking, or multi-layer annotation are needed — for example, the VS Code editor uses a similar concept for its DocumentHighlight feature, introduced in June 2023. Before this API, developers had to split text nodes or wrap spans around every range, which caused reflow, broken selection, and accessibility problems.

The API was originally proposed in July 2021 and reached full interoperability by early 2024. Today, applying five independent highlight layers to a 50,000-word document takes under 2 ms, compared to minutes for an equivalent DOM manipulation approach. "It changes how we think about text presentation entirely," wrote one contributor.
